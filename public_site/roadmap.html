  1. Public Alpha Rehearsal Evidence v0: implemented as supervised local/static evidence using existing safe-mode checks; no deployment was performed.
  2. LIVE_ALPHA_01 Production Public-Alpha Wrapper: implemented as a localhost-default public-alpha wrapper and config check; no deployment, no live source probes, and no production approval.
  3. Public Publication Plane Contracts v0: implemented as route, data, client profile, deployment target, redirect, and base-path governance before deployment.
  4. GitHub Pages Deployment Enablement v0: implemented as a static-only workflow and artifact check for publishing public_site; deployment success is unverified until GitHub Actions evidence exists.
  5. Static Site Generation Migration v0: implemented as a stdlib-only generator that writes site/dist while public_site remains the deployable artifact.
  6. Generated Public Data Summaries v0: implemented as deterministic static /data summaries from governed inventories and eval/audit sources; no live API, probes, or external observations.
  7. Lite/Text/Files Seed Surfaces v0: implemented as static no-JS/no-download compatibility seed surfaces under lite/, text/, and files/.
  8. Static Resolver Demo Snapshots v0: implemented as fixture-backed static no-JS demo snapshots under demo/; no live search, API, backend hosting, or external observations.
  9. Custom Domain / Alternate Host Readiness v0: implemented as provider-neutral static-host readiness inventory, base-path policy, operator checklist, and validator; no DNS, CNAME, alternate-host deployment, provider config, or backend hosting.
  10. Live Backend Handoff Contract v0: implemented as a contract-only /api/v1 reservation, capability flag, and error-envelope layer; no backend is hosted and /api/v1 is not live.
  11. Live Probe Gateway Contract v0: implemented as disabled-by-default source probe policy, caps, cache/evidence requirements, and operator gates; no probes or network calls.
  12. Rust Query Planner Parity Candidate v0: implemented as an isolated Rust planner candidate against expanded Python-oracle goldens; Python remains the oracle and Rust is not wired into runtime.
  13. Compatibility Surface Strategy v0: implemented as cross-surface strategy, capability matrix, route matrix, and old-client/native/snapshot/relay readiness policy; no new runtime behavior.
  14. Signed Snapshot Format v0: implemented as a static/offline snapshot format contract, checksum policy, signature-placeholder policy, and deterministic repo seed example under snapshots/examples/static_snapshot_v0; no production signing keys, executable downloads, public /snapshots/ route, relay runtime, or native runtime.
  15. Relay Surface Design v0: records future operator-controlled local/LAN relay policy only; no relay runtime, sockets, FTP, SMB, WebDAV, Gopher, proxy, backend, private data exposure, write/admin route, live-probe passthrough, or native runtime is implemented here.
  16. Rust Source Registry Parity Catch-up v0: recommended next non-network Codex milestone to update the isolated Rust source-registry candidate against the expanded Python source/capability shape.
  17. Manual Observation Batch 0 Execution: human-operated external baseline observations for selected query/system slots; remains parallel manual work.
  18. Internet Archive Live Probe v0: future only, after explicit human approval, live probe gateway review, and source-policy review; not queued for unattended implementation.
