# Security Policy

Eureka does not currently operate a production hosted service. GitHub Pages, if
enabled by an operator, is a static-only publication surface and does not run
the Python backend.

Do not submit secrets, credentials, API keys, private local paths, private
caches, raw local indexes, executable payloads, installers, or copyrighted
payload dumps in issues, pull requests, packs, examples, or fixtures.

For sensitive security issues, prefer GitHub Security Advisories if they are
available for this repository. If no private GitHub advisory path is available,
use a private maintainer contact only if one is documented by the repository
owner. A private contact address is not yet documented and remains pending.

Public issues are acceptable for non-sensitive bugs, documentation drift,
validator failures, and reproducible safety-policy gaps that do not expose
private data or exploit details.

Current non-implemented surfaces include live probes, downloads, uploads,
accounts, hosted contribution intake, AI runtime, and production backend
hosting. The security policy must be updated before any hosted public service
or live-source capability is enabled.

This file is a minimal pre-production placeholder, not a complete vulnerability
disclosure, incident response, privacy, takedown, backup, or operations policy.
