# Typed AI Output Contract v0

Typed AI Output Contract v0 defines the shape of future AI output records. The
contract keeps model output in the right lane: a suggestion or review
candidate, never canonical truth.

`contracts/ai/typed_ai_output.v0.json` validates the output envelope. It does
not implement model calls, provider runtime, prompt logging, telemetry, AI
search, embeddings, vector retrieval, or evidence acceptance.

## Output Envelope

Every typed output includes:

- `output_id`
- `provider_ref`
- `task_type`
- `output_type`
- `status`
- optional `subject_ref`
- optional `source_refs`
- optional `evidence_refs`
- optional `generated_text`
- optional `structured_claims`
- `limitations`
- `required_review: true`
- `prohibited_uses`
- `created_by_provider`
- `notes`

The example outputs under
`examples/ai_providers/disabled_stub_provider_v0/examples/` are hand-authored
synthetic fixtures. They are not generated by a model call.

## Allowed Output Types

Allowed output types are:

- `interpreted_query_candidate`
- `alias_candidate`
- `metadata_claim_candidate`
- `compatibility_claim_candidate`
- `review_claim_candidate`
- `member_path_candidate`
- `source_match_candidate`
- `identity_match_candidate`
- `explanation_draft`
- `ocr_cleanup_candidate`
- `absence_explanation_draft`
- `contribution_draft_candidate`

Structured claims may align with evidence/contribution pack claim language, but
remain candidates until reviewed.

## Required Review

Every output must include:

- `required_review: true`
- `prohibited_uses` containing `canonical_truth`, `rights_clearance`,
  `malware_safety`, and `automatic_acceptance`
- limitations explaining uncertainty
- provider provenance

AI output can support future contribution drafting or review triage only after
separate tooling validates it. It cannot mutate the runtime, local index,
source registry, public search, or master index.

## Evidence Links

Outputs should link to source or evidence references where possible. Missing
evidence does not make the output true; it increases review burden. Conflicts
must be preserved rather than merged away.

## Not Implemented

Typed AI Output Contract v0 does not implement an output ingestion runtime,
model calls, prompt capture, output cache, public-search display, contribution
pack drafting, review queue submission, master-index mutation, rights
clearance, malware safety, or production AI support.
