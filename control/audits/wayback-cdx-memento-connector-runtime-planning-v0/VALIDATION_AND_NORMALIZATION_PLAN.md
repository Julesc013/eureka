# Validation And Normalization Plan

Future runtime must validate and normalize:

- Response shape.
- URI-R.
- Capture timestamps.
- Status codes.
- MIME types.
- Digest/checksum metadata summaries.
- Duplicate capture summaries.
- Nearest capture summaries.
- Time-map summaries.
- Source attribution.
- Error envelopes.
- Raw payload exclusion.
- Archived content exclusion.
- Private URI redaction.

No validation/normalization runtime is implemented by P88.
