# Evidence-Weighted Factor Implementation

Strong evidence:

- exact identifier, checksum, or intrinsic ID match
- explicit evidence refs
- reviewed provenance labels

Medium evidence:

- recorded fixture metadata
- source-backed public-safe metadata

Weak evidence:

- lexical, name, or alias-only match

Negative or informational labels:

- candidate-only status
- conflict present
- source gap
- capability gap
- evidence missing
- rights/risk caution

Evidence is not truth. Source posture is not trust. Candidate confidence is not acceptance. Missing evidence remains visible in the factor list and user-visible reasons.
