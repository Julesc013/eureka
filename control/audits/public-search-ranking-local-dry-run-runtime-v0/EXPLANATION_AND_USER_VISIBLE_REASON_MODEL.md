# Explanation And User-Visible Reason Model

Every proposed dry-run ordering decision includes:

- a public reason
- an audit reason
- visible match, evidence, source, compatibility, gap, action, and rights/risk caveats when available

Rules:

- No hidden score.
- No raw private query.
- No model-generated answer.
- No rights clearance claim.
- No malware safety claim.
- No installability claim.
- If a reason cannot be generated, current-order fallback is required.
