# Search Need Record Schema

Required fields cover schema identity, lifecycle status, need identity, target
object, originating inputs, aggregate summary, source and capability gaps,
checked/not-checked scope, evidence/result context, future-only next steps,
priority, privacy, retention, aggregation, limitations, no-mutation guarantees,
and notes.

Hard no-mutation fields are false for master index, local index, candidate
index, probe enqueueing, result cache, miss ledger, query observation,
telemetry, external calls, and public need runtime creation.
