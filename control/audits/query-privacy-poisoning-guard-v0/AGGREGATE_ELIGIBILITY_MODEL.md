# Aggregate Eligibility Model

Public aggregate eligibility is allowed only after privacy and poisoning risk review. Allowed aggregate fields are coarse public-safe fields such as normalized query terms, intent, object kind, platform, artifact type, source family, miss type, gap type, probe kind, candidate type, or absence status. Raw queries, IP addresses, account identifiers, private paths, secrets, private URLs, local identifiers, and user file names are disallowed.
