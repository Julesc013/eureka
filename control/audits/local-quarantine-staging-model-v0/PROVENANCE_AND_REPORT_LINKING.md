# Provenance And Report Linking

Future staged state must link to:

- pack root or pack ID
- pack version
- pack checksum
- validate-only import report ID
- validation timestamp if available
- validator versions or tool IDs
- source/evidence/index/contribution pack references
- operator or user staging action when a future tool exists
- limitations and notes

User account identity is not required in v0. Provenance links do not make
staged records true, public-safe, rights-cleared, malware-safe, or accepted by
the master index.
