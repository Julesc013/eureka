# Risks And Limitations

- Validation success can be mistaken for truth, evidence, rights clearance,
  malware safety, or review acceptance.
- The validator checks shape and conservative safety boundaries; it does not
  evaluate factual correctness.
- It does not evaluate model quality because no model is called.
- It does not inspect arbitrary directories or user systems.
- Runtime AI integration still requires explicit future approval, consent,
  credential policy, logging policy, cache policy, and review workflow design.
