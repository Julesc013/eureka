# Operator Steps

Required human/operator steps to verify GitHub Pages:

1. Open repository Settings -> Pages.
2. Set Build and deployment Source to GitHub Actions.
3. Save the Pages setting.
4. Go to Actions -> Deploy static public site to GitHub Pages.
5. Rerun the latest failed workflow or dispatch it manually.
6. Confirm the workflow succeeds through Configure Pages, artifact upload, and
   deploy steps.
7. Record the deployed URL, workflow run URL, run ID, deployed commit SHA, and
   timestamp.
8. Open the deployed root page.
9. Open the deployed status page.
10. Open the deployed sources page.
11. Open the deployed search handoff page.
12. Open deployed lite, text, and files surfaces.
13. Record screenshot or text evidence if desired.
14. Update a future evidence audit with the run URL and deployed commit.

These steps are not completed by P52. They remain operator-gated.
