# Risks And Limitations

Known limitations:

- no runtime import
- no local indexing
- no upload or hosted contribution intake
- no master-index acceptance
- no executable plugin model
- no live connector behavior
- no binary artifact policy
- no production signing
- no malware scanning
- no rights clearance
- no broad source coverage claim

Source packs can make contribution safer, but they can also carry bad metadata
or overbroad claims. Future acceptance paths need human or governed review,
quarantine rules, public/private classification, and provenance checks before
any public index uses them.
