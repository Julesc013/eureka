# Risks And Limitations

Remaining risks:

- future remote providers can leak private data without explicit policy
- model output can be mistaken for evidence truth
- generated explanations can overstate source coverage
- provider identity and model version changes require invalidation policy
- AI-assisted contribution drafting needs review queue discipline

Limitations:

- schemas are structural contracts, not runtime enforcement
- no model output validator beyond examples exists yet
- no runtime consent, credential, or logging UI exists
- no AI-assisted pack/contribution workflow exists
