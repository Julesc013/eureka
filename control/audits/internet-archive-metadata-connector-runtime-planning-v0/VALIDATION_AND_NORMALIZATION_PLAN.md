# Validation And Normalization Plan

Future runtime must validate and normalize:

- Response shape.
- Item identifier shape.
- Bounded query shape.
- Metadata fields.
- Dates.
- Collection and media-type values.
- File-listing summaries.
- Checksum/fixity extracted only from metadata summaries.
- Source attribution.
- Error envelopes.
- Raw payload exclusion.
- Private data redaction.

No validation/normalization runtime is implemented by P87.
