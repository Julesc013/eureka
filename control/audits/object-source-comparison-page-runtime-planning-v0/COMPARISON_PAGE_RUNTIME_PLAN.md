# Comparison Page Runtime Plan

Input:

- comparison_id from public index or reviewed comparison record only
- optionally future explicit compare request only after separate contract and safety gate

Output:

- HTML page
- text/lite page
- JSON API response

Sections:

- subjects
- criteria
- matrix
- identity/version/representation/source/evidence/compatibility comparisons
- rights/risk/action comparison
- conflicts/disagreements
- absence/gaps
- limitations

Constraints:

- no arbitrary compare of unreviewed objects
- no live source calls
- no winner without scoped evidence
- no candidate promotion
- no mutation
