# Candidate Type Taxonomy

P64 candidate types include:

- object identity
- software version
- source record
- evidence record
- representation
- member path
- compatibility claim
- checksum
- release metadata
- package metadata
- source match
- identity match
- alias
- absence
- extraction
- OCR text
- query interpretation
- actionability
- unknown

Allowed future destinations are labels only. They do not route records to
source cache, evidence ledger, contribution packs, review queue, or master
index in P64.
