# Candidate Lifecycle Model

`contracts/query/candidate_lifecycle.v0.json` defines:

- `draft_example`
- `dry_run_validated`
- `observed_future`
- `normalized_future`
- `candidate_future`
- `review_required`
- `quarantined`
- `rejected_future`
- `superseded_future`
- `promoted_future`

`promoted_future` is vocabulary only. P64 examples keep promotion disabled,
accepted truth false, and master-index mutation false.
