# File Stability Decisions

| File | Stability class | Intended consumers | Allowed client use | Prohibited client use | Breaking-change sensitivity | Current gaps | Next guard |
| --- | --- | --- | --- | --- | --- | --- | --- |
| `site_manifest.json` | `stable_draft` for high-level flags, `experimental` for nested policy/surface details | Static pages, snapshots, relay/native planning, audit tools | Read schema, generator, static-artifact identity, public data file list, no-live/no-deployment flags, and base-path targets. | Do not treat nested policy objects as a production API, deployment proof, or runtime capability map. | Changing high-level static/no-live flags or removing `public_data_files` requires a breaking-change note. | Nested policy records are copied from several evolving governance inventories. | Generated Artifact Drift Guard v0. |
| `page_registry.json` | `stable_draft` for route basics, `experimental` for extra metadata | Static clients, file-tree/snapshot consumers, future native/relay planners | Display route path, title, status, stability, JavaScript requirement, base-path portability, and static-hosting posture. | Do not assume reserved routes are implemented; do not branch on repo `source_file` paths. | Removing or renaming implemented route basics requires a breaking-change note. | Future route metadata may expand as live backend/static split evolves. | Route/data contract drift guard. |
| `source_summary.json` | `stable_draft` for source IDs/status/coverage basics, `experimental` for capability detail booleans | Static clients, snapshots, future relay/native clients, Rust parity docs | Display `source_id`, label, family, status, posture, coverage depth, placeholder/future markers, and no-live flags. | Do not treat individual capability booleans as finalized connector contracts or live support. | Removing source IDs or changing placeholder/live semantics requires a breaking-change note. | Capability detail shape may evolve with source coverage expansion. | Source/public-data field drift guard. |
| `eval_summary.json` | `experimental` overall, `stable_draft` for no-live/no-external-observation flags | Static status pages, audits, future dashboards | Display current local eval/search/manual-baseline posture with explicit pre-alpha limitations. | Do not use counts as production benchmarks, external-baseline observations, or stable quality metrics. | Schema removals require notes; count value movement is non-breaking. | Search-usefulness categories remain active audit taxonomy. | Eval summary drift guard after audit taxonomy changes. |
| `route_summary.json` | `stable_draft` for count keys and public-alpha non-production flag, `experimental` for route detail arrays | Public-alpha review, static status pages, future operators | Display route posture counts and review/blocked summaries. | Do not treat as GitHub Pages routing, production API discovery, or route implementation proof. | Removing route-count keys or `public_alpha_not_production` requires a breaking-change note. | Details depend on evolving local route inventory. | Public-alpha route/data drift guard. |
| `build_manifest.json` | `volatile` diagnostic file with selected `stable_draft` safety flags | CI/static artifact checks, audit tools | Check schema, generator, artifact root, listed public data files, and disabled behavior flags. | Do not depend on `commit`, `built_at`, `source_inputs`, or validation-command lists as durable client API. | Removing disabled-behavior flags requires a breaking-change note; provenance values are volatile. | CI substitutions remain placeholders in committed artifacts. | Generated Artifact Drift Guard v0. |

No generated public data file is production-stable. The current promise is
pre-alpha `stable_draft` for specific field paths only.
