# Mutation And Promotion Boundary

- validate-only import may produce reports
- quarantine may stage candidate material in future local store only after approval
- promotion is separate from import
- source cache mutation requires source-cache runtime approval
- evidence ledger mutation requires evidence-ledger runtime approval
- candidate index mutation requires candidate runtime approval
- public index mutation requires index rebuild/promotion approval
- master index mutation requires master index review queue and explicit review
- import runtime must not bypass promotion policy
