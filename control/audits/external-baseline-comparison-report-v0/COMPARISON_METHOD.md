# Comparison Method

Comparison dimensions for future observed records:

- query/task match
- exact artifact found
- useful partial result
- source coverage
- compatibility evidence
- representation/member access
- provenance/evidence clarity
- absence explanation
- action safety
- user effort
- result freshness
- not comparable reason

Allowed comparison labels: `eureka_better`, `baseline_better`, `both_good`, `both_partial`, `eureka_only`, `baseline_only`, `neither`, `not_comparable`, and `insufficient_data`.

This pack avoids wins language unless explicit scoped evidence supports it.
