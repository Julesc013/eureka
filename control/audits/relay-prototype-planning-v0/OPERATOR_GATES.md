# Operator Gates

Future implementation requires all gates below:

- explicit human approval
- prototype path/name approval
- bind scope approval
- input roots approval
- privacy review
- action policy review
- local cache/privacy policy review
- snapshot consumer review
- test plan approval
- rollback/disable path
- no private data confirmation
- no live probe confirmation
- no live backend proxy confirmation
- no download/install automation confirmation

The default approved scope should be localhost-only and read-only. LAN exposure,
public exposure, private data, live API use, live probes, protocol translation,
or executable/download behavior requires separate future approval.
