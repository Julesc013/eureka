# Code of Conduct

Eureka work should be respectful, evidence-focused, and careful with user,
source, and preservation contexts.

Participants are expected to:

- Be constructive and specific.
- Respect boundaries between facts, candidates, plans, and opinions.
- Avoid harassment, personal attacks, discriminatory language, and deliberate
  disruption.
- Avoid publishing private data, credentials, exploit details, copyrighted
  payload dumps, executable payloads, or unreviewed sensitive material.
- Keep disagreement grounded in repo evidence, reproducible command output, and
  documented project doctrine.

For non-sensitive conduct concerns, contact a maintainer or open a GitHub issue
with appropriate context. For private or sensitive conduct concerns, use a
private maintainer contact only if one is documented by the repository owner.
That private contact path is not yet documented and remains pending.

This is a concise pre-production conduct standard. It is not a complete legal,
employment, moderation, or hosted-community policy.
